# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A zero-build browser app — a real-time meeting assistant for a Thai dev team. The user listens to a meeting via Web Speech API, optionally shares their screen, and gets streamed answers from an LLM. UI text is Thai; the target user is a Go / PostgreSQL / React+TypeScript backend dev (stack is configurable, not hardcoded).

**No build step / bundler.** `index.html` (HTML shell + the small Tailwind config inline `<script>`), `styles.css` (all component CSS), and the app logic split into **ordered classic scripts** under `js/` — loaded in numeric order, sharing one global scope (top-level `const`/`let`/`function` are globals; cross-file refs resolve at call-time, so load order only matters for top-level *executed* code, which is arranged so each file's deps load before it):

- `js/01-config.js` — `PROVIDERS`, `QA_SYSTEM`/`EST_SYSTEM`/`CORRECT_SYSTEM`, `stackLine`
- `js/02-state.js` — `store`/`skey`/`getKey`, state vars, DOM refs (`$`), model-select helpers, `loadProvider`
- `js/03-models.js` — `modelsReq`, `fetchModels`, provider/model handlers, `loadProvider()` call, `checkAudioDevices`
- `js/04-voice.js` — mode/`setMode`, toggles, `refreshStatus`, screen share, voice listen toggle (`startWebListen`/`stopWebListen`/`voiceSend`; `stopListen` = dead/compat)
- `js/05-llm.js` — `buildRequest`, `streamLLM`
- `js/06-render.js` — `el`/`esc`/`highlightCode`/`inline`/`mdToHtml`/`renderEstimate`, chat bubbles, voice preview, `correctText`, `looseJSON`
- `js/07-main.js` — `submit`, sessions/routing, font, init IIFE
- `js/08-overlay.js` — หน้าต่างลอย (floating overlay) โหมดเดียว: **Document PiP** (ลอยเหนือแอปอื่น always-on-top / ซ่อนตอนแชร์เฉพาะหน้าต่าง ไม่ใช่ทั้งจอ; ต้อง Chrome/Edge 116+). mirror `#results` เข้าหน้าต่าง PiP ด้วย `MutationObserver` ตัวเดียว (`startMirror`/`syncOverlay`/`stopMirror`). เปิดตอนกด `startBtn` (`openOverlay`→`openPip`), ปิดตอนกลับ home (`closeOverlay`→`closePip`); ปุ่ม `🪟 #floatBtn` ใน chat header = toggle เปิด/ปิด (`toggleOverlay`/`updateFloatBtn`/`isOverlayOpen` — เช็ค `pipWin && !pipWin.closed`). `openPip` guard เปิดซ้ำ (เปิดอยู่แล้ว → `focus()` ไม่สร้าง window ที่ 2); `requestWindow` reject → `updateFloatBtn()` ใน catch; `pagehide` ทิ้ง state เฉพาะตอน `pipWin.closed` จริง (กัน bfcache/navigate race). PiP คัด stylesheet ผ่าน `copyStylesTo` (same-origin clone cssRules, cross-origin/Tailwind CDN → `<link>`). มี control bar (`.float-composer`: `.fc-mic`/`.fc-stop`/`.fc-screen`/`.fc-input`/`.fc-send`) ในแผง PiP — proxy ไปปุ่มหลัก (`micBtn`/`stopBtn`/`screenBtn`) ด้วย `.click()` + `submit()`; `wireFloatControls` ผูก event, `syncFloatControls` คัด label/state จากปุ่มหลัก ขับด้วย `ctrlObserver` (MutationObserver บนปุ่มหลัก — connect ใน `openPip` ผ่าน `startCtrlObserver`, disconnect ใน `closePip`/`pagehide` ผ่าน `stopCtrlObserver`). **(loads last)**

No package.json, no framework, no dependencies. Serve over `http://` (Chrome/Edge — required for `SpeechRecognition`; media APIs are blocked on `file://`); LLM calls go directly from the browser to the provider. Optional `config.local.js` (loaded before `js/*`) sets `window.__MA_KEYS__` for local key auto-fill.

## Architecture

Key pieces (file in parens):

- **Multi-provider LLM abstraction** (`PROVIDERS`, `buildRequest`, `modelsReq`, `streamLLM`): the one part worth understanding. `buildRequest(provider, key, model, opts)` returns `{url, headers, body, extract}` normalizing four provider APIs (OpenRouter, Gemini, OpenAI, Claude) that have *different* request shapes, auth headers, image-embedding formats, and SSE response schemas. `extract(parsedJSON)` pulls the text delta out of each provider's distinct stream chunk. `buildRequest` takes a `think` opt (from the `🧠 คิดก่อนตอบ` toggle `thinkOn`, persisted `ma_think`, default **off**): when off it disables model reasoning — gemini `generationConfig.thinkingConfig.thinkingBudget:0`, openrouter `reasoning:{enabled:false}` (claude sends no thinking anyway; openai left as-is). qa/est pass `think:thinkOn`; correctText + STT stay no-think. `streamLLM` is provider-agnostic — it reads the SSE `data:` lines and calls `extract` per chunk; it also accumulates **token usage** via each request's `usage(chunk)` parser into `req.usageAcc` ({in,out}) — gemini `usageMetadata`, claude `message_start`/`message_delta`, openai/openrouter final chunk (`stream_options:{include_usage:true}`). `submit()` reads `req.usageAcc` → `addCost()` attaches a `$` cost **only for OpenRouter** (`modelPrice()` uses the real `pin`/`pout` $/token captured into `modelsList` by `modelsReq` from OpenRouter's `pricing`; other providers have no per-model pricing so token-only) → stores `item.tok` ({in,out,cost?}) + shows a per-answer `.tok` badge (`tokBadge`/`fmtTok`/`fmtCost` in js/06) under the bubble; `updateTokTotal()` sums `curSess.items[].tok` (+cost) into the header `#tok`. `modelsReq(provider, key)` is the parallel abstraction for the `GET …/models` endpoints — its `extract(json)` returns a vision-filtered list of model ids that the ↻ button loads into the model `<select>` (cached in `localStorage` as `ma_models_<provider>`; falls back to `PROVIDERS[p].models` presets on fetch failure). **To add a provider, edit `PROVIDERS`, `buildRequest`, and `modelsReq`.**
- **Two modes** driven by `mode` state: `qa` (free-form answer, streamed token-by-token into a card) and `est` (work estimation — forces JSON output via the provider's json mode, parsed by `looseJSON` and rendered by `renderEstimate`). Each mode has its own system prompt: `QA_SYSTEM`, `EST_SYSTEM`. These set tone and are polyglot (no hardcoded stack). **Stack/context is a configurable field** (`#stackCtx` on home, persisted `ma_stack`, default empty) — `stackLine()` appends it to the qa/est system prompt at submit; empty = no pinned language/stack (answer per the question), filled = bias toward that stack but follow the question if it names another. **Mode is chosen on the home/lobby before starting (the `#homeModes` `.mode-card` picker → `homeMode`; `startBtn` calls `setMode(homeMode)` then `newSession()`) and is LOCKED per session — there are no composer tabs.** The chat header shows the current mode via the `#curMode` badge (`MODE_LABEL`). `setMode(m)` updates `mode`, the badge, mic label, send-button text, and input placeholder. The session's mode is persisted (`curSess.mode` in `ensureSession`/`saveItem`) and restored on reload (`initSessions` → `setMode(curSess.mode||"qa")`).
- **Voice loop** (`buildRec`, `startRec`, `voiceSend`): continuous `SpeechRecognition`; on final transcript + auto-send, a silence timer (`silenceMs`) triggers `voiceSend`. Listening is a **2-state toggle mirroring the gemini mode** (`micOn` + `setMicUI`): idle (`🎤 เริ่มฟัง`) ⇄ listening (`⏹ หยุด` + `✂️ ส่งเลย` `#stopBtn`). `startWebListen()`/`stopWebListen()` start/stop the recognizer **non-terminally** — stop keeps `finalText`+preview and does NOT lock the composer; pressing start again resumes/accumulates (buffer is cleared by `voiceSend` on send and by `resetListen` on newSession). The `#stopBtn` (`✂️ ส่งเลย`) calls `voiceSend()` to send the current `finalText` immediately without waiting for silence, then keeps listening (`if(micOn) startRec()`). `resetListen()` (called by `newSession`) returns to idle. `onend` restarts only while `micOn`. **(`stopListen()` — the old terminal `⏹ จบ` that set `stopped=true` + `lockComposer` view-only — has no button anymore; it and the `stopped`/`lockComposer`/`#roNote` + `initSessions` restore path are kept only for backward-compat with old persisted `curSess.stopped` sessions.)** The recognizer is stopped during the LLM call and restarted in `onend` to avoid feedback. The transcribed text is shown **live as a chat bubble** (`updateVoicePreview`/`clearVoicePreview` → `.voice-live` preview, "🗣 คนถาม (กำลังพูด…)"), NOT in the textarea — accumulated in `finalText`; `submit()` falls back to `finalText` when the textarea is empty (typed text wins). The preview is cleared on submit / mic-stop / newSession / clear.
- **STT engine toggle** (`sttEngine`, persisted `ma_stt`, default `web`): home settings seg (`#sttWeb`/`#sttGemini`). (Gemini Live realtime STT was removed — v1beta WS auto-detects language, so Thai+English code-switching came out wrong-language/garbled; the batch `gemini` mode below is the accurate path.) `web` = Web Speech (live, the state machine above). `gemini` = **สด+AI auto-cut** (hands-free continuous) via `toggleGeminiRecord` (js/04-voice.js): press once `🎤 เริ่มฟัง` → `⏹ หยุด`. It runs **both** `MediaRecorder` (per-utterance clip) **and** Web Speech (`hybridRec=true`): `rec.onresult` shows the live preview AND acts as **VAD** — after `silenceMs` of silence it calls `cutHybridClip()` which `mediaRec.stop()`s the current clip. `_hybridClipDone` then immediately `startHybridClip()`s the next clip (no gap), decodes the finished clip to mono WAV (`blobToWav`) and POSTs it as inline audio to `transcribeAudio` (js/05-llm.js) for an accurate transcript, and **always sends** it (hands-free) through a small serial queue (`hybridQ`/`_drainHybridQ`, guards the `busy` flag so utterances during an in-flight answer aren't dropped). Falls back to the Web Speech text (`hybridText`) if it fails. Stopping cuts+sends the final clip and closes the stream. **`transcribeAudio` picks its backend via `sttBackend()` (js/02): `provider==="openrouter"` with an OpenRouter key → POST to OpenRouter `chat/completions` with `model:"google/gemini-2.5-flash"` and the audio as an `input_audio` (base64 wav) content part — uses the user's OpenRouter credit, NO separate Gemini key; otherwise Gemini-native `generateContent` (needs a Gemini key — `provider==="gemini"` uses the main key field, others use the separate `#geminiKey` field that `syncGeminiKeyRow` reveals).** Shared verbatim/anti-hallucination prompt = `STT_PROMPT`; both paths post-strip a leading conversational preamble. The mode is gated by `sttBackend().key` (blocked + hinted if absent). Hands-free + accuracy; per-utterance transcribe lag (~1-2s after silence) and one audio call per utterance.
- **LLM transcript correction** (`CORRECT_SYSTEM`, `correctText`): ASR of Thai-mixed-English dev speech is error-prone. Optional `#correctBtn` (✨) toggle in the header (`correctVoice`, persisted `ma_correct`, default **off**) — when on, `voiceSend` runs `finalText` through the configured LLM (`correctText` → `streamLLM`, `maxTokens` scaled to input length) to fix mis-transcribed terms before submitting (adds one LLM call). Off = raw transcript sent as before.
- **Screen share** (`getDisplayMedia` → hidden `<video id="screenVideo">` → `captureFrame`): grabs a JPEG frame (downscaled to max 1280px) only at submit time when sharing is on, base64-encoded and attached to the request as an image. All four providers are vision-capable.
- **Layout & sessions (lobby → chat)**: no sidebar/navbar. `main.view` holds three sections toggled by `showView(name)`: `#viewHome` (lobby — the `#settings` panel + `▶ เริ่มแชท` `startBtn` + `#homeList` session list), `#viewCurrent` (`.chat-wrap`: sticky header with ← home/`#curMode` badge/title/clear, `#results` chat-msgs, `.composer` with voice/screen/input — no mode tabs), `#viewOld` (readonly chat). Conversation renders as **chat bubbles, all left-aligned** with a role label (`roleLabel()` 🗣 คนถาม / 🤖 AI) — `addUserMsg()` (teal bubble) + `addAiMsg()` (panel bubble, holds `.ans`); `scrollBottom()` after each. Cold controls (ไทย/Eng lang seg + ⚡ auto-send + ✨ correction) live in the chat **header**; hot controls (🎤 mic + 🖥 screen) stay in the composer (mode is fixed per session, picked on home). **Sessions persist in IndexedDB** (db `ma_db`, store `sessions`, keyPath `id`) — larger quota than localStorage (no ~5MB cap). Pattern: an in-memory map `SESS` (id→session) is the sync source of truth; `loadSess`/`sessIndex` read it synchronously, `persistSess`/`deleteSession` write memory + fire-and-forget `idbPut`/`idbDel` (helpers in js/02). `initSessions` is async — `await idbReady` → `SESS = await idbGetAll()` → one-time `migrateFromLocalStorage()` (imports legacy `ma_sess_*` keys if IDB is empty; old localStorage left as backup). `sessIndex()` is derived from `SESS` (no separate index key). If IndexedDB is unavailable, `idb*` no-op (sessions degrade, don't break). A session is `{id,title,mode,createdAt,updatedAt,provider,model,stopped?,items}`; an `item` is `{q,mode,hadImage,raw}` (raw = markdown / `JSON.stringify(est)`), rebuilt by `renderSessionInto` (chronological). `submit()`→`saveItem()`; `newSession`/`deleteSession`(+confirm)/`openSession`/`initSessions` (always lands on `home`/settings by default, restoring the last session into memory so clicking it reopens the chat). Settings live only on home. Font size = `--fs` slider, persisted `ma_fontsize`.
- **Tailwind**: loaded via Play CDN with `corePlugins.preflight:false` (so it does NOT reset the existing component CSS — `.ans`/markdown/forms/cards stay intact) and brand colors mapped in `tailwind.config`. Tailwind utilities are used for shell polish (gradient title, hover/shadow/transition); the component styling stays in `styles.css`. Needs internet on page load.
- **Persistence**: chat sessions → **IndexedDB** (see above). Settings — selected provider, per-provider model, fetched model cache, `ma_cur`/`ma_silence`/`ma_correct`/`ma_stt`/`ma_fontsize` — stay in `localStorage` (`ma_*` keys) via the `store` helper (small, sync). **API keys are kept in `sessionStorage` via the separate `skey` helper** (cleared when the tab closes); `getKey(p)` migrates any key found in legacy `localStorage` into `sessionStorage` and purges it. Keys never leave the browser except to the chosen provider. Claude calls use `anthropic-dangerous-direct-browser-access` to allow direct browser-to-API requests (CORS).

## Conventions

- UI copy and comments are Thai; identifiers and code are English.
- No external libs — keep it dependency-free and **no build step** (`index.html` + `styles.css` + `js/0N-*.js` ordered classic scripts). Don't add a bundler/framework unless there's a strong reason. Component styling goes in `styles.css`; app logic in the matching `js/` file. When adding a top-level call, make sure its deps live in an earlier-numbered file (or are function decls called only at runtime).
- DOM access via the `$(id)` helper; elements built with `el(tag, cls, html)`; always `esc()` user/LLM text before `innerHTML`.
