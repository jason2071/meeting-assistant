# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A zero-build browser app — a real-time meeting assistant for a Thai dev team. The user listens to a meeting via Web Speech API, optionally shares their screen, and gets streamed answers from an LLM. UI text is Thai; the target user is a Go / PostgreSQL / React+TypeScript backend dev (stack is configurable, not hardcoded).

**No build step / bundler.** `index.html` (HTML shell + the small Tailwind config inline `<script>`), `styles.css` (all component CSS), and the app logic split into **ordered classic scripts** under `js/` — loaded in numeric order, sharing one global scope (top-level `const`/`let`/`function` are globals; cross-file refs resolve at call-time, so load order only matters for top-level *executed* code, which is arranged so each file's deps load before it):

- `js/01-config.js` — `PROVIDERS`, `QA_SYSTEM`/`EST_SYSTEM`/`CORRECT_SYSTEM`, `stackLine`
- `js/02-state.js` — `store`/`skey`/`getKey`, state vars, DOM refs (`$`), model-select helpers, `loadProvider`
- `js/03-models.js` — `modelsReq`, `fetchModels`, provider/model handlers, `loadProvider()` call, `checkAudioDevices`
- `js/04-voice.js` — mode/`setMode`, toggles, `refreshStatus`, screen share, voice listen state machine (`startListen`/`pauseListen`/`resumeListen`/`stopListen`/`voiceSend`)
- `js/05-llm.js` — `buildRequest`, `streamLLM`
- `js/06-render.js` — `el`/`esc`/`highlightCode`/`inline`/`mdToHtml`/`renderEstimate`, chat bubbles, voice preview, `correctText`, `looseJSON`
- `js/07-main.js` — `submit`, sessions/routing, font, init IIFE (**loads last**)

No package.json, no framework, no dependencies. Serve over `http://` (Chrome/Edge — required for `SpeechRecognition`; media APIs are blocked on `file://`); LLM calls go directly from the browser to the provider. Optional `config.local.js` (loaded before `js/*`) sets `window.__MA_KEYS__` for local key auto-fill.

## Architecture

Key pieces (file in parens):

- **Multi-provider LLM abstraction** (`PROVIDERS`, `buildRequest`, `modelsReq`, `streamLLM`): the one part worth understanding. `buildRequest(provider, key, model, opts)` returns `{url, headers, body, extract}` normalizing four provider APIs (OpenRouter, Gemini, OpenAI, Claude) that have *different* request shapes, auth headers, image-embedding formats, and SSE response schemas. `extract(parsedJSON)` pulls the text delta out of each provider's distinct stream chunk. `streamLLM` is provider-agnostic — it reads the SSE `data:` lines and calls `extract` per chunk. `modelsReq(provider, key)` is the parallel abstraction for the `GET …/models` endpoints — its `extract(json)` returns a vision-filtered list of model ids that the ↻ button loads into the model `<select>` (cached in `localStorage` as `ma_models_<provider>`; falls back to `PROVIDERS[p].models` presets on fetch failure). **To add a provider, edit `PROVIDERS`, `buildRequest`, and `modelsReq`.**
- **Two modes** driven by `mode` state: `qa` (free-form answer, streamed token-by-token into a card) and `est` (work estimation — forces JSON output via the provider's json mode, parsed by `looseJSON` and rendered by `renderEstimate`). Each mode has its own system prompt: `QA_SYSTEM`, `EST_SYSTEM`. These set tone and are polyglot (no hardcoded stack). **Stack/context is a configurable field** (`#stackCtx` on home, persisted `ma_stack`, default empty) — `stackLine()` appends it to the qa/est system prompt at submit; empty = no pinned language/stack (answer per the question), filled = bias toward that stack but follow the question if it names another. **Mode is chosen on the home/lobby before starting (the `#homeModes` `.mode-card` picker → `homeMode`; `startBtn` calls `setMode(homeMode)` then `newSession()`) and is LOCKED per session — there are no composer tabs.** The chat header shows the current mode via the `#curMode` badge (`MODE_LABEL`). `setMode(m)` updates `mode`, the badge, mic label, send-button text, and input placeholder. The session's mode is persisted (`curSess.mode` in `ensureSession`/`saveItem`) and restored on reload (`initSessions` → `setMode(curSess.mode||"qa")`).
- **Voice loop** (`buildRec`, `startRec`, `voiceSend`): continuous `SpeechRecognition`; on final transcript + auto-send, a silence timer (`silenceMs`) triggers `voiceSend`. Listening is a 3-state machine (`micOn`/`paused` + `setMicUI`): idle (`🎤 เริ่มฟัง`) → listening (`⏸ พัก` + `⏹ จบ` `#stopBtn`) → paused (`▶ ฟังต่อ` + `⏹ จบ`). `pauseListen()` stops the recognizer but **keeps** `finalText`+preview so `resumeListen()` continues; `stopListen()` is terminal — clears the buffer, stops screen share, sets `stopped=true`, **hides the whole composer** (`lockComposer` → `#composer` hidden + `#roNote` "ดูได้อย่างเดียว") so the session becomes **view-only**, and persists `curSess.stopped` (a stopped session restores view-only on reload via `initSessions`). pause/resume keep the composer active; only stop locks it. Start a new session to listen again. `resetListen()` (called by `newSession`) returns to idle with buttons restored. `onend` restarts only while `micOn`. The recognizer is stopped during the LLM call and restarted in `onend` to avoid feedback. The transcribed text is shown **live as a chat bubble** (`updateVoicePreview`/`clearVoicePreview` → `.voice-live` preview, "🗣 คนถาม (กำลังพูด…)"), NOT in the textarea — accumulated in `finalText`; `submit()` falls back to `finalText` when the textarea is empty (typed text wins). The preview is cleared on submit / mic-stop / newSession / clear.
- **STT engine toggle** (`sttEngine`, persisted `ma_stt`, default `web`): home settings seg (`#sttWeb`/`#sttGemini`). `web` = Web Speech (live, the state machine above). `gemini` = **batch push-to-record** via `transcribeAudioGemini` (js/05-llm.js): the mic button becomes `🎤 เริ่มอัด` → `⏹ ถอดเสียง`; `toggleGeminiRecord` (js/04-voice.js) records with `MediaRecorder`, on stop decodes the blob to mono WAV (`blobToWav`) and POSTs it as inline audio to Gemini `generateContent` for a verbatim transcript, then feeds `finalText`/preview → auto-sends if `autoSend` else lights the send button. Requires a Gemini key (`getKey("gemini")`, independent of the chat provider); the toggle is blocked + hints if absent. More accurate for Thai+English than Web Speech, but not live (one clip per press) and costs an audio call.
- **LLM transcript correction** (`CORRECT_SYSTEM`, `correctText`): ASR of Thai-mixed-English dev speech is error-prone. Optional `#correctBtn` (✨) toggle in the header (`correctVoice`, persisted `ma_correct`, default **off**) — when on, `voiceSend` runs `finalText` through the configured LLM (`correctText` → `streamLLM`, `maxTokens` scaled to input length) to fix mis-transcribed terms before submitting (adds one LLM call). Off = raw transcript sent as before.
- **Screen share** (`getDisplayMedia` → hidden `<video id="screenVideo">` → `captureFrame`): grabs a JPEG frame (downscaled to max 1280px) only at submit time when sharing is on, base64-encoded and attached to the request as an image. All four providers are vision-capable.
- **Layout & sessions (lobby → chat)**: no sidebar/navbar. `main.view` holds three sections toggled by `showView(name)`: `#viewHome` (lobby — the `#settings` panel + `▶ เริ่มแชท` `startBtn` + `#homeList` session list), `#viewCurrent` (`.chat-wrap`: sticky header with ← home/`#curMode` badge/title/clear, `#results` chat-msgs, `.composer` with voice/screen/input — no mode tabs), `#viewOld` (readonly chat). Conversation renders as **chat bubbles, all left-aligned** with a role label (`roleLabel()` 🗣 คนถาม / 🤖 AI) — `addUserMsg()` (teal bubble) + `addAiMsg()` (panel bubble, holds `.ans`); `scrollBottom()` after each. Cold controls (ไทย/Eng lang seg + ⚡ auto-send + ✨ correction) live in the chat **header**; hot controls (🎤 mic + 🖥 screen) stay in the composer (mode is fixed per session, picked on home). Sessions persist in `localStorage`: index `ma_sessions` (each entry carries `mode` for the history badge) + per-session `ma_sess_<id>` ({id,title,mode,...,items}); an `item` is `{q,mode,hadImage,raw}` (raw = markdown / `JSON.stringify(est)`), rebuilt by `renderSessionInto` (chronological). `submit()`→`saveItem()`; `newSession`/`deleteSession`(+confirm)/`openSession`/`initSessions` (always lands on `home`/settings by default, restoring the last session into memory so clicking it reopens the chat). Settings live only on home. Font size = `--fs` slider, persisted `ma_fontsize`.
- **Tailwind**: loaded via Play CDN with `corePlugins.preflight:false` (so it does NOT reset the existing component CSS — `.ans`/markdown/forms/cards stay intact) and brand colors mapped in `tailwind.config`. Tailwind utilities are used for shell polish (gradient title, hover/shadow/transition); the component styling stays in `styles.css`. Needs internet on page load.
- **Persistence**: selected provider, per-provider model, and the fetched model cache live in `localStorage` (`ma_*` keys) via the `store` helper. **API keys are kept in `sessionStorage` via the separate `skey` helper** (cleared when the tab closes); `getKey(p)` migrates any key found in legacy `localStorage` into `sessionStorage` and purges it. Keys never leave the browser except to the chosen provider. Claude calls use `anthropic-dangerous-direct-browser-access` to allow direct browser-to-API requests (CORS).

## Conventions

- UI copy and comments are Thai; identifiers and code are English.
- No external libs — keep it dependency-free and **no build step** (`index.html` + `styles.css` + `js/0N-*.js` ordered classic scripts). Don't add a bundler/framework unless there's a strong reason. Component styling goes in `styles.css`; app logic in the matching `js/` file. When adding a top-level call, make sure its deps live in an earlier-numbered file (or are function decls called only at runtime).
- DOM access via the `$(id)` helper; elements built with `el(tag, cls, html)`; always `esc()` user/LLM text before `innerHTML`.
